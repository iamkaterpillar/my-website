Using AI agents in Cursor

When I first started building this website, I was using VS Code—but I wouldn’t say I was comfortable with it. Everything felt unfamiliar, and I was relying heavily on ChatGPT to walk me through each step.

After hearing a lot of people recommend Cursor, I figured it was worth upgrading and giving it a try. I liked the interface right away, but at first, I didn’t even think to explore the AI agents. I was mostly just using it like a smarter version of VS Code.

Recently though, I finally started digging into the AI features—and it’s been a game changer.

What is Cursor?

Cursor is an AI-native IDE (Integrated Development Environment). Think of it as VS Code, but with superpowers. It’s designed from the ground up to integrate AI into your workflow—whether that’s writing new code, debugging issues, or learning as you go.

How I’m using AI agents

Here’s how I’ve been using Cursor’s AI agents to build and improve this site:

  • Code cleanup: The AI helps tidy up formatting, catch inconsistencies, and suggest clearer patterns.
  • Feature implementation: When adding new features, it recommends best practices and even scaffolds out clean, efficient code.
  • Bug fixing: I can paste error messages or describe weird behavior, and it often points me directly to the fix.
  • Learning: The AI doesn't just give solutions—it explains them. That’s been huge for leveling up my understanding.

Recent improvements (thanks to Cursor)

Some tangible improvements I’ve made recently with the help of Cursor’s AI:

  • Reorganized the file structure for better clarity and scalability
  • Added meta tags for SEO and social previews
  • Implemented responsive design patterns for mobile friendliness
  • Created a reusable styling system using utility classes
  • Added social media icons using clean, lightweight SVGs

Tips for working with AI agents

If you’re new to using AI in your dev workflow, here are a few things I’ve learned:

  • Be specific: Vague prompts = vague results. Tell the AI exactly what you need.
  • Stay in control: Don’t blindly accept suggestions—use them as a starting point, and verify everything.
  • Ask “Why?”: Let the AI teach you. It’s a surprisingly good mentor when prompted to explain.
  • Start small: Try it on low-risk tasks first. You’ll build trust over time.
