The tools we used to build Bebop

When you’re building a product as fast as we were at Bebop, you don’t always stop to document every tool, integration, or dependency you pulled in along the way. You just… keep shipping.

But now that I’m nearing the end of my time with the team, I wanted to pause and actually write it all down—partly so I don’t forget, and partly because there’s a lot here I’d want to reuse (or avoid) next time.

This is the stack we built Bebop with—from the code editor to the chain integrations to the dashboards we checked every morning.

How we worked together

First up: collaboration. A lot of this is standard startup fare, but a few things—like Gather or 1Password—were surprisingly critical to keeping the team aligned.

  • Slack: Where everything happened (and occasionally too much).
  • Google Workspace: Docs, Sheets, Slides, Calendar—classic, and still essential.
  • 1Password: Shared credentials, secure notes, API keys… a lifesaver.
  • Confluence + Jira: Product specs and sprint boards. Not everyone’s favorite, but it worked.
  • GitBook: Public-facing docs and guides.
  • Figma + Storybook: For design handoff and UI consistency.
  • Gather: We actually used this. It made remote work feel a little less isolating.
  • Tunnelblick (VPN): For secure access when needed.

What we built with

On the dev side, we leaned heavily into TypeScript, React, and Next.js—with a modern Web3 stack layered on top.

  • GitHub: For repo management, of course.
  • Cursor / VS Code: Each dev had their own preferences.
  • Vercel: Preview links for every PR—huge for async reviews.
  • Playwright + Cypress: For E2E testing, depending on the flow.
  • PostHog + Google Analytics: We started with GA, then moved to PostHog for product feature tracking.
  • DBeaver: Great for poking around in databases (historicasl trades, tokens, etc.)
  • Grafana: For system-level monitoring.
  • React, TypeScript, Next.js, Wagmi, Viem, RainbowKit: Our front-end foundation.
  • Web3Modal / ConnectKit: To make wallet connection as smooth as possible.

API integrations

This is where things got fun—and occasionally messy. We stitched together data and functionality from a ton of sources.

  • Intercom: In-app chat support and help pages.
  • Alchemy / QuickNode / Helius: Node providers for chain data and RPCs.
  • Coinbase: For wallet creation and management.
  • Across: We didn't integrate but I would have loved to for bridging in-app!
  • ENS + clusters: For usernames within the app and for features like swap & send.
  • GoPlus + Webacy: For token safety and risk warnings.
  • CoinGecko API: Token prices, metadata, and icons.
  • Dune API: Powered our portfolio dashboards.
  • TradingView: For candlestick charts on token pairs.
  • Cloudflare: CDN and security layer for the frontend.

How we did marketing

We were a small team, so everything from memes to press to newsletters ran through the same stack.

  • Medium: Where we originally published long-form posts.
  • Beehiiv: For newsletters (but was moved to Twitter).
  • Twitter: Still our biggest community channel by far.
  • LinkedIn: Occasionally useful. Mostly ignored.
  • Farcaster: We tried. Didn’t stick.

Why I wrote this

This isn’t a pitch or a product update. It’s just a snapshot—for myself, really. When I move on from Bebop, I want to remember the stack we built, the tools that helped (and the ones that didn’t), and what I’d reach for again in future roles.

If you’re building something similar, I hope this helps you shortcut a few decisions.
