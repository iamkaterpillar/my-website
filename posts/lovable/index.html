Vibe coding with Lovable

I recently built my personal portfolio site — katfore.com — using a tool called Lovable. It took less than a day from idea to live site, and I didn't even have to write code to get started. Here's how it all worked.

What is Lovable?

Lovable is an AI-powered app builder that turns natural language prompts into full-stack applications. You describe what you want — "a portfolio site with a hero section, project grid, and contact form" — and Lovable builds it. You can preview the design, iterate, connect GitHub, and deploy, all within the tool. It's fast, beginner-friendly, and surprisingly powerful.

Building my site

I knew I wanted a portfolio — something quick to share with recruiters and collaborators. I bought the domain katfore.com (yay, it was available!), created my content, and prompted Lovable with the layout I wanted. Within minutes, I had a working design.

I connected Lovable to GitHub and created a new repository: github.com/iamkaterpillar/portfolio-site. Since the free version of Lovable doesn't allow direct code edits, I cloned the repo locally and used Cursor to make custom changes.

Switching between Lovable, GitHub, and Cursor did cause some versioning headaches — pushing local changes could conflict with anything edited via Lovable afterward. Worth noting if you're planning to go hybrid like I did.

Hosting with Vercel

Instead of hosting with GitHub Pages like my other site, I gave Vercel a try. We use it at Bebop for PR preview links, so I already knew it worked well. Setup was seamless: I connected my GitHub repo to Vercel, deployed the site, and now whenever I redeploy from Lovable, the live site updates instantly.

What I liked about Lovable

  • Natural language interface — no blank page syndrome
  • Instant iterations — you can go from sketch to site in minutes
  • GitHub integration
  • Frontend and backend support (auth, database, endpoints)
  • Great starting point for MVPs or mockups

What I'd improve

  • Free users can't edit code inside Lovable
  • Syncing between GitHub, local edits, and Lovable versions can get messy
  • You'll want to replace the default Lovable branding (favicon, share image, etc.)

Final thoughts

If you've ever wanted to launch a site but felt blocked by the code, Lovable + Vercel is an incredible combo. You can go from an idea in your head to a live website in under an hour — seriously.

Check out my site here, and if you're curious to try it yourself, start with Lovable.
