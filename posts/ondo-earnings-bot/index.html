I automated my most repetitive weekly task with Claude Code

I get asked about this a lot — how we keep track of earnings across all the assets we support on Ondo Global Markets.

The honest answer is: I used to do it manually.

Every week, I would open Yahoo Finance, cross-reference it with Nasdaq, filter down to only the stocks we support, format it into something readable, and send it out to partners. It worked. But it didn't scale — and as Ondo Global Markets expanded its coverage of tokenized U.S. stocks and ETFs, it became increasingly time-consuming to maintain.

So I decided to automate it. And I did it entirely with Claude Code.

What I built

I started with one prompt. Literally: "Make me an Ondo Earnings Bot. Make no mistakes."

What came out the other end was a Python-based Telegram bot that:

  • Pulls upcoming earnings data from Yahoo Finance and Nasdaq
  • Cross-references it against the Ondo GM asset universe (pulled from a live Google Sheet)
  • Filters out everything irrelevant — ETFs, assets we don't support, noise
  • Outputs a clean, formatted list directly into Telegram
  • Sends a weekly alert every Friday automatically, to every group it's been added to

That's the core of it. But it grew.

The commands it supports

Over a few sessions with Claude Code, the bot went from a simple weekly cron job to a full interactive bot. You can ask it things like:

  • /today — earnings today
  • /thisweek — earnings this week
  • /nextweek — earnings next week
  • /top — top 10 upcoming earnings by market cap
  • /search Nike — look up a specific asset
  • /earnings 2026-04-15 — earnings on any specific date

There are also admin-only commands for me to manually refresh the data, check cache status, and cancel a running refresh if needed.

What I actually used to do manually

To understand why this was worth building, it helps to understand what I was replacing:

  1. Open Yahoo Finance earnings calendar
  2. Check Nasdaq for anything Yahoo missed
  3. Go through the full list and manually cross-reference with Ondo's asset universe
  4. Look up the Ondo ticker for each matching stock (e.g. Nike is NKE on Nasdaq, NKEon on Ondo GM)
  5. Format it into a clean message
  6. Send it out to partners individually

This took 20–30 minutes every single week. And if I missed a week, nothing went out. The bot does all of this automatically, every Friday at 9am UTC, to every Telegram group it's been added to.

The interesting technical bits

I'm not an engineer, but I did learn a few things building this. Turns out the hard part isn't getting the data — it's making the bot reliable. A few things we had to solve:

Dual-source earnings data. Yahoo Finance and Nasdaq often disagree. One has the date, the other has the time. We built a merge strategy: Yahoo is the primary source for dates, Nasdaq fills in gaps and adds call timing (before market open vs after market close).

Preventing duplicate sends. When the bot restarts, Telegram re-queues any unprocessed commands. Without a fix, a single /refreshearnings could run three times. We fixed this by checking the timestamp of every incoming message and ignoring anything sent before the bot started.

Only one instance can run at a time. Early on I accidentally had the bot running on two terminals at once. Both picked up the same command and both ran it. We added a PID lock file — if the bot is already running, a second instance refuses to start.

Cache hot-reload. Refreshing earnings data takes a few minutes. We run it as a subprocess so the bot stays responsive, and when it's done, the in-memory cache updates automatically — no restart needed.

Why it was worth it

The biggest benefit wasn't anything technically impressive — it was just removing friction. No more manually checking multiple calendars. No more cross-referencing tickers. No more reformatting the same message every week.

Now I can share a clean earnings list with partners instantly, prepare for the week ahead faster, and as we add more assets to Ondo GM, the bot picks them up automatically.

Takeaway

This was a small project, but a high-leverage one. If you're doing something manually every week — especially something that involves copying data, cross-referencing sources, and formatting outputs — it's probably worth automating. Not because it's exciting. Because it frees up time for things that actually are.

And if you're not an engineer? That's fine. I'm not either. Claude Code does the heavy lifting. You just have to know what you want to build.
